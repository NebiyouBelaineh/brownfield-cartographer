## Architecture Overview
dbt-core is a data transformation framework that classifies and executes DAG-based operations on SQL models, analyses, tests, and functions through a centralized node typing system defined in `node_types.py`. The architecture flows from source data (raw tables and SQL definitions) through the core execution engine, which uses domain-specific exception handling (`exceptions.py`) to manage errors during transformation pipelines. Pytest fixtures in the test suite provide isolated project environments to validate the system's behavior across functional scenarios. Telemetry tracking (`tracking.py`) wraps Snowplow to collect anonymous usage statistics, running parallel to the main execution path. Ultimately, the system transforms input data sources into output artifacts (models like `metricflow_time_spine`, test results, and computed functions), with comprehensive error handling and observability integrated throughout the pipeline.
