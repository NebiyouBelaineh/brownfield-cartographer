## Architecture Overview
This system is a data orchestration and transformation pipeline built on Dagster that manages B2B organization data workflows, with dbt integration for SQL-based transformations. The architecture consists of three main layers: (1) a PostgreSQL event logging system that tracks Dagster asset execution and lineage, (2) a project orchestration layer (`dg_projects`) that defines data dependencies and asset configurations, and (3) dbt automation tooling that generates and maintains data transformation models from source database schemas. Data flows from external sources (edxorg, mitxonline) through dbt-generated models and Python transformation assets, with intermediate deduplication and reconciliation steps handling data integrity issues like course partition mismatches, ultimately producing cleaned datasets for downstream B2B analytics consumption. The system includes deployment-specific reconciliation scripts that correct historical data issues and ensure consistency across partitioned assets.
